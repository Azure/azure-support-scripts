RHUI Check Multi-Image Test Lab Builder

About RHUI Check Test Lab Builder

  • Purpose: Generates an ARM template that deploys one control VM plus one VM per RHEL image in the catalog, for automated RHUI break/check/fix validation testing.
  • Control VM: A RHEL 9 Gen2 Standard_B2s VM with Ansible, httpd, and a key server. It receives rhui-check.py and the test playbook via cloud-init. After deployment, it polls all image VMs with ping (30 forks) and runs the playbook once all VMs are online.
  • rhui-check.py: By default, the latest upstream rhui-check.py is downloaded from GitHub on page load (if the editor is empty). You can still paste or edit it manually. The script is gzip-compressed and base64-encoded into custom data, then written to /home/ansible/rhui-check.py on the control VM. The playbook copies it to each target.
  • Test playbook: The RHUI break/check/fix playbook YAML. It is gzip-compressed and base64-encoded into cloud-init custom data, then auto-executed by the control VM after all VMs respond to ping.
  • Force images: Pick specific RHEL images from the catalog instead of deploying all of them.
  • Quota guard & multi-region: Set a regional vCPU limit (default 100). When projected usage exceeds the limit, VMs are automatically split across multiple regions with hub-spoke VNet peering.
  • Output: Single-region: resource-group-scope ARM template. Multi-region: subscription-scope template. Includes VNet/subnet/NSG, control VM with public IP, and one VM per selected image with SSH key auth.
  • Results: The control VM runs httpd on port 80. After the playbook completes, results are served at /results.json and an HTML validation report at /index.html.
